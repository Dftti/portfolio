Key Features

Every feature designed for the reality of live, paid events.

🔄

Anti-Rapid-Switching

Current playlist must play at least 3 songs before switching. Rolling vote windows smooth out spikes. 3-minute cooldown after each switch.

🚫

Duplicate Prevention

Tracks every queued song. Songs played in the last 2 hours are blocked. Falls back to runner-up playlist if all tracks are duplicates.

🔧

Admin Override

One cell in Google Sheet controls everything. Set a playlist name to force it, type PAUSE to stop, clear it to resume voting.

🛡️

Error Handling

Every API call is wrapped. If Spotify disconnects, the admin gets a Telegram alert. If Sheets is down, cached data keeps the music going.

📊

Real-Time Logs

Terminal shows every vote count, playlist switch, and queued song with timestamps. Full visibility during the event.

🎵

Always 3-5 in Queue

Queue manager checks every 30 seconds and tops up automatically. No gaps, no silence, no interruptions.

The Core Loop

Clean, readable, maintainable. One file, one loop.

# spotify_dj.py — core architecture

POLL_INTERVAL = 30          # seconds between cycles
MIN_QUEUE     = 3           # minimum songs in queue
TARGET_QUEUE  = 5           # fill queue up to this
MIN_SONGS_BEFORE_SWITCH = 3 # anti-rapid-switching

while True:
    # 1. Count votes from Google Sheet
    votes  = read_votes(sheets)
    winner = get_winning_playlist(votes)

    # 2. Check admin override
    override = check_admin_override(sheets)

    # 3. Anti-switching: only change after 3+ songs
    if winner != active and songs_played >= 3:
        active = winner
        log("🔄 Switched to: {winner}")

    # 4. Keep queue at 3-5 songs
    queue = spotify.queue()
    if len(queue) < MIN_QUEUE:
        tracks = get_tracks(active)
        tracks = filter_duplicates(tracks)
        for track in tracks[:TARGET_QUEUE - len(queue)]:
            spotify.add_to_queue(track)
            log("🎵 Queued: {track.name}")

    time.sleep(POLL_INTERVAL)
